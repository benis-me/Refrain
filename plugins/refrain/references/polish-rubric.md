# Polish Rubric

Use this rubric to review Refrain outputs and projects that inherit it.

## 1. Taste

Score high when the product is restrained, specific, and operational.

Look for:

- one clear accent
- stable spacing
- borders over shadows
- dense but calm tool surfaces
- concrete product language
- no generic AI visual tropes

Fail when the output looks like a generic generated SaaS page.

## 2. Agent Usability

Score high when a future agent can act without interpretation.

Look for:

- clear triggers
- defaults
- forbidden moves
- exact commands
- evidence requirements
- small files with obvious ownership

Fail when guidance is inspirational but not executable.

## 3. Engineering Restraint

Score high when the implementation is small, local-first, and inspectable.

Look for:

- direct interfaces
- dependency-light baseline
- explicit runtime event types
- tests at risky boundaries
- no premature framework or plugin layer

Fail when abstraction hides the behavior an agent needs to verify.

## 4. Verification

Score high when claims are backed by fresh output.

Look for:

- root verification
- template verification
- visible-surface checks for UI work
- clear report of gaps

Fail when completion is inferred from code edits alone.

## 5. Portability

Score high when the artifact can be copied into a new project without source
project context.

Look for:

- no source-project path dependency
- no private runtime assumptions
- clear install or copy instructions
- local defaults

Fail when a future project needs hidden context to use the system.
