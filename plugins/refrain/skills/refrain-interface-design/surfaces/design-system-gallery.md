# Design System Gallery

A design system is a selectable product object, not a color dropdown.

## Gallery

- searchable list
- built-in/custom split
- visual cards with cover, mark, swatches, and component specimen
- category and short style signal
- selected state that is easy to scan

## Detail

- overview
- colors
- typography
- spacing and radii
- components
- brand examples
- light/dark or on-surface samples when relevant

## Agent Contract

Every system should expose guidance an agent can apply: atmosphere, token roles,
type rules, component treatment, layout rules, depth rules, do/don't, responsive
behavior, and prompt guidance.

## Reject

- plain text theme lists
- swatches without components
- style copy without implementation consequences
