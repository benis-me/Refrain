# Creation Home

A creation home starts the product's real first workflow. It is not a marketing
hero.

## Required Material

- actual composer or input surface
- attach/reference affordance when source material matters
- mode selector when output depth changes
- design-system selector when style matters
- runtime or agent target when execution can vary
- recent work or resumable objects
- honest empty/error state

## Layout

Keep the composer visually central but not oversized. Recent work should be
visible enough that continuation feels natural.

## Reject

- oversized slogan as the first screen
- generic feature cards
- hidden primary input
- empty screen that waits for perfect onboarding copy
