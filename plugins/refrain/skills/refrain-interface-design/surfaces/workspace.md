# Workspace

A workspace is the product's main operating room.

## Required Material

- global navigation
- primary object identity
- main work plane
- preview or result surface when output is visual
- files, versions, history, quality, logs, or inspector when relevant
- run state near the command that started it
- recovery path for failed or stale work

## Layout

Use full-bleed work planes and panes. Keep decoration out of the frame; let the
object, state, and controls carry the screen.

## Verification

Check desktop and narrow widths. Resizing panes must not hide required actions or
leave blank ambiguous regions.

## Reject

- dashboard summary replacing the work
- marketing layout inside project pages
- tabs that do not map to real peer views
