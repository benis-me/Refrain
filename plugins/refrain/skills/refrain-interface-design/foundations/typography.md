# Typography

Type carries hierarchy before color or decoration.

## Roles

- display: restrained scale, strong weight, no gradient clipping
- heading: clear section ownership, not hero-sized inside tools
- body: concrete copy, comfortable line height
- metadata: compact monospace or tracked uppercase labels
- numerics: tabular when values compare

## Rules

- Use uppercase only with tracking.
- Keep button and control text short enough for compact layouts.
- Prefer concrete nouns and verbs over product explanation.
- Reserve large type for actual first-viewport identity, not every panel.

## Reject

- generic display font hardcoding
- text justification
- multiple h1s in app shells
- filler claims such as "unlock", "supercharge", or "seamless"
